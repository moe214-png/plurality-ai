# AI 对话可视化面板 - 使用指南

## 🎨 面板特性

✨ **实时展示**
- 左侧: Claude 的所有回应
- 右侧: Codex 的所有回应
- 每 2 秒自动刷新一次

📊 **详细信息**
- 总消息数统计
- 最后发言者
- 实时更新时间戳
- Claude vs Codex 消息数对比

## 🚀 快速开始

### 第一步：启动面板

直接双击：`start_panel.bat`

你会看到：
```
🚀 启动 AI 对话可视化面板
==================================================

打开浏览器访问:
  本机: http://127.0.0.1:5000
  手机/局域网: http://192.168.x.x:5000

按 Ctrl+C 停止服务器
```

### 第二步：打开浏览器

电脑上在浏览器地址栏输入：
```
http://127.0.0.1:5000
```

手机上使用同一个 Wi-Fi，然后在手机浏览器输入启动窗口里显示的 `手机/局域网` 地址，例如：
```
http://192.168.1.23:5000
```

或者自动打开：
```
start http://127.0.0.1:5000
```

如果手机打不开，通常是 Windows 防火墙拦截了 Python。允许 Python 访问专用网络，或在防火墙里放行 TCP 端口 `5000`。

### 第三步：并排看对话

## 💡 完整工作流

### 1. 启动所有组件

终端 1 - 启动自动化系统：
```bash
cd d:\AI对话
python auto_dialogue.py
# 选择 "1" 启动监控
```

终端 2 - 启动可视化面板：
```bash
cd d:\AI对话
python panel.py
```

浏览器 - 打开面板：
```
http://127.0.0.1:5000
```

### 2. 初始化对话

编辑或创建 `awaiting_claude.txt`，输入初始问题：
```
分析这段代码的优化空间...
```

### 3. Claude 处理

在 VS Code 中：
1. 打开 `awaiting_claude.txt`
2. 让 Claude 分析并生成回应
3. 将完整回应复制到 `claude_response.txt`
4. **保存文件**

面板会立即显示 Claude 的回应在左侧！

### 4. Codex 处理

系统已自动在 `awaiting_codex.txt` 生成提示。

在 Codex 桌面端：
1. 打开 `awaiting_codex.txt`
2. 复制内容到 Codex
3. 让 Codex 继续分析
4. 将回应复制到 `codex_response.txt`
5. **保存文件**

面板会立即显示 Codex 的回应在右侧！

### 5. 持续对话

系统会自动循环，你只需在面板上看对话进行。

## 🌐 给不同局域网的朋友使用

不同局域网不能使用 `192.168.x.x` 地址。分享前建议先在 `.env` 里设置访问密码：

```env
PANEL_USER=panel
PANEL_PASSWORD=换成一个长一点的密码
```

改完后重启 `start_panel.bat`。浏览器会弹出登录框，用户名填 `panel`，密码填你设置的值。

推荐两种分享方式：

1. **Tailscale / ZeroTier 私有组网**：适合长期给固定朋友使用。你和朋友都安装并加入同一个私有网络，朋友访问你的虚拟 IP 加 `5000` 端口。
2. **Cloudflare Tunnel / ngrok 临时公网链接**：适合临时分享。把公网链接转发到本机 `http://127.0.0.1:5000`。

不建议直接在路由器做端口转发。这个面板会消耗你的 API key 配额，并且能查看、导出、清空对话记录。

## 🐛 故障排除

### 无法连接到面板
```
"无法访问 127.0.0.1:5000"
```
解决：
1. 检查 `start_panel.bat` 是否还在运行
2. 端口可能被占用，尝试修改 `panel.py` 的端口号

### 手机无法打开面板
解决：
1. 确认手机和电脑连接的是同一个 Wi-Fi
2. 不要在手机上输入 `127.0.0.1`，要输入启动窗口显示的 `手机/局域网` 地址
3. 允许 Windows 防火墙放行 Python，或放行 TCP 端口 `5000`
4. 如果电脑正在使用 VPN、代理或公共网络，先临时关闭或切换到家庭/专用网络再试

### 面板不刷新
解决：
1. 刷新浏览器 (F5)
2. 检查 `dialogue_log.json` 是否存在
3. 确保对话系统在运行

### 消息显示为空
解决：
1. 检查是否有新消息被保存
2. 点击 **🔄 立即刷新** 按钮
3. 检查 `auto_dialogue.py` 是否在监控文件

## 📝 自定义面板

要修改刷新频率，编辑 `panel.py`：

```python
REFRESH_INTERVAL = 2000  # 改为其他毫秒数，如 1000 (1秒)
```

要修改端口号，编辑 `panel.py`：

```python
PORT = 5001
```

也可以在启动前设置环境变量：

```bat
set PANEL_PORT=5001
python panel.py
```

如果只想限制为电脑本机访问，可以设置：

```bat
set PANEL_HOST=127.0.0.1
python panel.py
```
